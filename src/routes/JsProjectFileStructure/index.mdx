# Javascript Project's Files
- So.. Today we gonna look at javascript projects file structure and their workings , When you work with javascript libraries or frameworks you usaully have some repitive files that needs in your project to able run to run that specific technology..    

# 1. Package.json
![](/packagejson.png)
It is a json file that stores your project's metadata (name, version, description) , libraries used , frameworks and other dependecies info.. contains their version , Defines scripts etc etc 
It's the most important file in your project cause without it you cannot run your project will tell why..

```json
{
  "name": "example-with-solidbase",  
//   - name of your project 
//   - Rules :
//       Lowercase, no spaces (use - or _ )
//       Must be unique if publishing to npm
//   - Required : Optional for private application; Yes (if publishing to npm)

  "version" : 19.3.0 ,
//  Current version of your project, npm requires it but local projects can omit it.
//  Format: MAJOR.MINOR.PATCH (e.g., "1.0.0")
//  Required : Optional for private apps; Yes (if publishing)


  "description": "A social media application",
//  Short summary of your project
//  Required : Nope. Optional


   "main": "dist/index.js",
//  Entry point of your app (when someone import your package) , not used for personal project cause you manually runs it.
//  Example: import c from "packageName" ; so whenever they import it gonna point to your file mentioned in your main field
//  dist is your compiled project , it is how you publish or upload your package/application
//  Required : Only when your project is published as a library/package

  

  "type": "module",
//  type of your project either es6 or es7 
//  Required : optional 



  "scripts": {
    "dev": "vinxi dev",
    "build": "vinxi build",
    "start": "vinxi start",
    "version": "vinxi version"
  },
  "dependencies": {
    "@iconify-json/ri": "^1.2.5",
    "@kobalte/solidbase": "^0.2.11",
    "@solidjs/meta": "^0.29.4",
    "@solidjs/router": "^0.15.3",
    "@solidjs/start": "^1.1.0",
    "@tailwindcss/vite": "^4.1.13",
    "solid-js": "^1.9.5",
    "tailwindcss": "^4.1.13",
    "vinxi": "^0.5.6"
  },
  "engines": {
    "node": ">=22"
  }
}
```
